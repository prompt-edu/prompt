# Database Migrations & Queries

Each microservice owns a separate PostgreSQL database. Schema changes use `golang-migrate`; Go is
generated by `sqlc`. For the full workflow use the **`sqlc-migration`** skill.

- **Migrations:** `servers/<service>/db/migration/`, sequentially numbered `NNNN_description.up.sql`
  (zero-padded 4 digits). Auto-applied on server startup via `runMigrations()`.
- **Down migrations are optional** and inconsistent across services — only add a matching
  `NNNN_description.down.sql` if that service already has `.down.sql` files; otherwise stay up-only.
- Never edit a merged migration; add a new numbered one.
- **Queries:** `servers/<service>/db/query/*.sql` with sqlc annotations (`-- name: X :many|:one|:exec`).
- After any schema/query change run `sqlc generate` (or `make sqlc-<service>`) and commit the
  regenerated `db/sqlc/` together with the change.
- The `detect-migrations.yml` CI workflow inspects migration changes — keep the `NNNN_*.up.sql`
  naming exact.
