# sqlc Usage (Go services)

Type-safe DB access is generated by `sqlc` from `db/query/*.sql` + `db/migration/*.sql`. Config is
`servers/<service>/sqlc.yaml` (engine postgresql, pgx/v5, output `db/sqlc`, package `db`).

For writing migrations and queries and the regeneration workflow, use the **`sqlc-migration`**
skill. In Go code, use generated methods:

```go
import db "github.com/prompt-edu/prompt/servers/<service>/db/sqlc"
tutors, err := queries.GetTutors(ctx, coursePhaseID)
```

- Regenerate with `sqlc generate` (in the service dir) or `make sqlc-<service>`.
- Commit regenerated `db/sqlc/` together with the migration/query change.
- Each microservice has its own PostgreSQL database — never cross-query another service's DB.
